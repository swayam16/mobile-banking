# NovaPay — Mobile Banking App UX Redesign

A full end-to-end UX case study and interactive prototype for a fintech mobile banking application.

## Overview

NovaPay had a broken onboarding flow, a confusing transfer experience, and zero bill visibility. This project documents the full redesign process — from user research to a 60-screen high-fidelity prototype — with measurable outcomes.

## What's Inside

- **Interactive phone mockup** — tap through Home, Send Money, and Bill Pay screens
- **Research findings** — insights from 20+ user interviews and competitive analysis
- **Competitive analysis** — NovaPay vs. Chase, Revolut, and Monzo
- **Before / After breakdowns** — specific design decisions with rationale
- **Usability test results** — 3 rounds of Maze testing, 12 participants each
- **Measured outcomes** — drop-off rates, task completion, SUS scores

## Key Results

| Metric | Before | After |
|--------|--------|-------|
| Onboarding drop-off | 78% | 33% (−45%) |
| Transfer task time | ~4 min | ~1.9 min (2.1×) |
| Task success rate | 61% | 92% |
| Usability score (SUS) | — | 4.7 / 5 |

## Design Decisions

**Progressive disclosure** — Reduced onboarding from 11 steps to 5 by showing one task per screen and deferring KYC until the user hits a transaction limit.

**Contextual error messages** — Replaced generic "Something went wrong" errors with specific, actionable guidance (e.g., "Your selfie was blurry — try in better lighting").

**Bill visibility on home** — Surfaced upcoming bills directly on the home screen, eliminating a 3-level navigation dig.

**Quick actions** — Promoted Send, Request, Pay Bill, and Top Up to a persistent shortcut row above the transaction feed.
