Research phase

Understanding the real problems

Before designing a single pixel, I ran 20+ user interviews, a competitive analysis of leading fintech apps, and affinity mapping sessions to uncover key friction points.

78%
Abandoned onboarding
Users dropped off during the 11-step ID verification flow — most at step 4 (selfie capture).
3.2×
More taps for a transfer
Compared to Revolut, NovaPay required 3.2× more interactions to complete a simple P2P transfer.
91%
Wanted bill visibility
Users wanted to see upcoming bills and payment dates on the home screen, not buried 3 levels deep.

Design decisions

Progressive disclosure redesign

The core UX strategy was progressive disclosure — showing only what users need at each step, reducing cognitive load and guiding them forward with confidence.

Before
  • 11-step onboarding with all fields on one screen
  • ID upload + selfie + bank linking in sequence
  • No progress indicator — users felt lost
  • Generic error messages ("Something went wrong")
  • Transfer flow buried under 3 navigation levels
After
  • 5-step onboarding with one task per screen
  • Deferred KYC — users start with limited features
  • Step indicator + estimated time remaining
  • Contextual, specific error messages with fixes
  • Send button promoted to home screen quick actions
1
Phone + name
30 seconds. Just the essentials — phone number verification via SMS.
2
Set a passcode
Users choose biometric or 6-digit PIN. Immediate sense of security and ownership.
3
Explore app
Let users in with a $50 limit. KYC is prompted contextually when they hit the limit.
